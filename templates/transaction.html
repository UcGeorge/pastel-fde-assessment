Transaction Monitoring

Submit a financial transaction to Sigma's instant AI monitoring engine. The engine analyses the transaction payload in real time, cross-referencing behavioural patterns, device context, and user history to return a recommended action and risk score.

Transaction Details

Required

Core fields describing the financial transaction being submitted for analysis.

Your unique transaction ID for traceability.
Transaction value in the selected currency.
When the transaction occurred (local time).
The medium through which the transaction was made.
Direction of the transaction relative to the account.
Completion state of the transaction.
Whether funds moved outside your platform.

Anonymized User Context

Required

Hashed/tokenized identifiers for the initiating user. No PII required — Sigma only needs anonymized signals.

Your internal hashed user identifier.
Whether this user is restricted on your platform.
Whether the user has completed KYC verification.

Optional Metadata

Optional

Additional signals that improve Sigma's detection accuracy. The more context provided, the richer the risk analysis.

Account Flags

Transaction Context

Sender's account number or identifier.
Account balance prior to this transaction.
Description or memo on the transaction.
Helps correlate multiple actions in one session.
Value-Added Service recipient (airtime, cable, etc.).

User Profile

The user's originating IP at time of transaction.

Device Information

Optional

Device fingerprinting helps Sigma detect anomalous login and payment patterns.

Location Context

Optional

Geographic signals allow detection of impossible travel or high-risk region Activity.

Third-Party User Data

Optional

Data from the counterparty (receiver's) system, if available. Enables cross-platform risk correlation.

Account Limits

Optional

Declared limits on the account. Helps Sigma flag transactions that approach or breach thresholds.

Inline Screening

Optional

Sender/receiver names to run PEP & sanctions screening in-line with the transaction check.

Beneficiary Information

Optional

Metadata about who is receiving the funds. Helps detect new or unregistered counterparties.
